    Deadlock Avoidance: The L1 (Client) node must stall/block any new Channel A requests if there is a pending Channel B Probe.

    Data: Data width is 512 bits (64-byte cache line). Address width is 32 bits.

Personal Note:
    
    In TileLink, an L1 Cache (the Client) never talks directly to another L1 Cache. It only talks to the L2 Cache (the Manager). The L2 acts as the central directory.
    If Core 0 wants a cache line, it asks the L2.
    If the L2 knows Core 1 has modified that line, the L2 sends a "Probe" (Channel B) to Core 1.
    Core 1 writes the data back to the L2 (Channel C).
    The L2 then grants the data to Core 0 (Channel D).

    None (N): You have no permissions. This is the Invalid (I) state.
    Branch (B): You have read-only permissions. This is the Shared (S) state.
    Trunk (T): You have read/write ownership. This maps to the Modified (M) or Exclusive (E) state.
